# 建筑组高亮使用说明

## 问题描述
当建筑由多个方块组成时，使用ClickableObject只能高亮单个物体，无法高亮整个建筑。

## 解决方案
使用 ClickableBuildingGroup 脚本，可以高亮整个建筑的所有子物体。

## 使用步骤

### 1. 组织建筑结构
确保你的建筑层级结构如下：
```
Building (父物体)
├── Cube1 (子物体)
├── Cube2 (子物体)
├── Cube3 (子物体)
└── ...
```

### 2. 设置父物体
- 选择建筑的父物体（最顶层的空物体或主体）
- 添加 Box Collider 组件
- 调整 Collider 的 Size 覆盖整个建筑
- 可以调整 Center 来对齐 Collider 位置

### 3. 添加脚本
- 移除旧的 ClickableObject 脚本（如果有）
- 添加 ClickableBuildingGroup 脚本
- 配置参数：
  * Target Scene Name: 目标场景名称
  * Target Scene Index: 或使用场景索引（-1表示使用名称）
  * Hover Color: 悬停时的高亮颜色
  * Enable Highlight: 启用高亮效果（勾选）
  * Highlight Children: 高亮所有子物体（勾选）

### 4. 测试
- 运行游戏
- 鼠标移动到建筑上，整个建筑应该变色
- 点击建筑，触发场景切换

## 重要提示

### Collider设置
- Collider必须在父物体上
- Collider的大小要覆盖整个建筑
- 可以在Scene视图中看到黄色线框（Collider范围）

### 材质要求
- 所有子物体必须有Renderer组件
- 建议使用相同或相似的材质
- 如果某些子物体不想高亮，可以取消勾选Highlight Children

### 性能优化
- 脚本会自动创建材质副本
- 材质副本会在物体销毁时自动清理
- 如果建筑很复杂（超过50个子物体），可能会影响性能

## 对比：ClickableObject vs ClickableBuildingGroup

### ClickableObject（旧）
- 只能高亮单个物体
- 适合简单的单一物体
- 性能更好

### ClickableBuildingGroup（新）
- 可以高亮整个建筑组
- 支持多个子物体
- 适合复杂建筑
- 稍微消耗更多性能

## 常见问题

### 问题1：只有部分子物体高亮
- 检查是否所有子物体都有Renderer组件
- 确保Highlight Children被勾选
- 查看Console是否有警告信息

### 问题2：点击没反应
- 确保父物体有Collider
- 检查Collider的大小是否覆盖建筑
- 确保Collider没有被禁用

### 问题3：高亮颜色不明显
- 调整Hover Color，使用更鲜艳的颜色
- 建议使用：黄色(1, 1, 0.5)或橙色(1, 0.5, 0)
- 确保与建筑原色有足够对比度

### 问题4：鼠标移开后颜色没恢复
- 检查Console是否有错误
- 确保材质没有被其他脚本修改
- 尝试重新运行游戏

## 高级用法

### 自定义高亮范围
如果只想高亮部分子物体：
1. 取消勾选 Highlight Children
2. 只在父物体上添加Renderer
3. 或者创建一个中间层级结构

### 多个建筑
可以在场景中添加多个建筑，每个都使用ClickableBuildingGroup：
- Building1 → Scene2
- Building2 → Scene3
- Building3 → Scene4

### 调试Collider范围
- 选中建筑父物体
- 在Scene视图中会看到黄色线框
- 这就是Collider的实际范围
- 确保它覆盖了整个建筑

## 迁移指南

如果你已经使用了ClickableObject：

1. 选择建筑父物体
2. 移除 ClickableObject 组件
3. 添加 ClickableBuildingGroup 组件
4. 复制相同的场景设置
5. 勾选 Highlight Children
6. 测试效果
