# 场景加载问题解决方案

## 问题描述
出现以下两个错误：
1. Scene 'House' couldn't be loaded because it has not been added to the active build profile
2. NullReferenceException: Object reference not set to an instance of an object

## 问题原因
1. House 场景文件存在，但没有被添加到 Build Settings 中
2. 当场景加载失败时，代码没有检查 AsyncOperation 是否为 null，导致空引用异常

## 解决方案

### 方法一：使用自动工具（推荐）
1. 在 Unity 编辑器中，点击菜单：Tools > 自动添加所有场景到 Build Settings
2. 这会自动将 Assets/Scenes 文件夹中的所有场景添加到 Build Settings
3. 完成后可以通过 Tools > 查看 Build Settings 中的场景 来确认

### 方法二：手动添加
1. 在 Unity 编辑器中，打开 File > Build Settings
2. 点击 "Add Open Scenes" 按钮，或者
3. 将 Assets/Scenes 文件夹中的场景文件拖拽到 "Scenes In Build" 列表中
4. 确保所有需要的场景都被勾选（启用）

### 方法三：使用 Build Profiles（Unity 6 新功能）
1. 打开 File > Build Profiles
2. 在 Shared Scenes 或当前 Profile 的 Scenes 列表中添加场景
3. 确保场景被正确配置

## 已修复的代码改进

### SceneTransitionManager.cs
- 添加了场景存在性检查
- 添加了 AsyncOperation 空值检查
- 添加了详细的错误日志
- 当加载失败时会自动淡入回到当前场景，避免黑屏

### BuildSettingsHelper.cs（新增）
- 提供了自动添加场景的编辑器工具
- 位置：Assets/Scripts/Editor/BuildSettingsHelper.cs
- 菜单位置：Tools > 自动添加所有场景到 Build Settings

## 验证修复
1. 使用工具添加场景后，尝试运行游戏
2. 点击会触发场景切换的物体
3. 如果仍有问题，查看 Console 中的详细错误信息

## 注意事项
- 每次添加新场景后，都需要将其添加到 Build Settings
- 场景名称必须与代码中使用的名称完全一致（区分大小写）
- 建议使用场景索引而不是场景名称来加载场景，更加可靠
